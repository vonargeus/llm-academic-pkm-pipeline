# Summary

This paper introduces [[Ragas]] (Retrieval Augmented Generation Assessment), a framework designed for the [[reference-free evaluation]] of [[Retrieval-Augmented Generation]] (RAG) pipelines. RAG systems combine retrieval and generation modules to ground [[Large Language Models]] in external knowledge, but evaluating them is challenging without human-annotated ground truth. To solve this, the authors propose a suite of metrics—specifically targeting [[faithfulness]], answer relevance, and [[context relevance]]—using LLM-assisted prompting. Their main finding is that these automated, reference-free metrics can evaluate different dimensions of RAG performance effectively without relying on ground truth human annotations, enabling faster and more cost-effective evaluation cycles.

# Research Problem

Evaluating [[Retrieval-Augmented Generation]] (RAG) architectures is highly challenging because it requires assessing multiple distinct dimensions, such as retrieval quality and generation accuracy, simultaneously. Traditional evaluation methods often rely on measuring [[perplexity]] on a reference corpus or using short extractive question-answering datasets, which do not reflect real-world usage or align well with downstream performance. Furthermore, many existing evaluation strategies require access to token probabilities, which are unavailable for closed-source models like ChatGPT and GPT-4. This paper addresses the gap in the literature for a fully automated, [[reference-free evaluation]] framework that can assess RAG pipelines without requiring expensive human-annotated ground truth datasets.

# Method

The authors present [[Ragas]], a framework that evaluates RAG pipelines across three key dimensions: [[faithfulness]], answer relevance, and [[context relevance]]. The evaluation process is designed to be reference-free, meaning it does not require human-annotated gold-standard answers. Instead, it leverages a powerful LLM (specifically `gpt-3.5-turbo-16k` in their implementation) to act as an evaluator through structured [[prompting strategies]].

To measure [[faithfulness]] (ensuring the generated answer is grounded in the retrieved context and free of [[hallucination detection|hallucinations]]), [[Ragas]] uses a two-step prompting process. First, the LLM decomposes the generated answer into a set of individual, focused statements. Second, the LLM verifies whether each statement is supported by the retrieved context, outputting a binary verdict (Yes/No) along with a brief explanation. The final faithfulness score is calculated as the ratio of supported statements to the total number of generated statements.

The framework also targets answer relevance (ensuring the generated answer addresses the user's initial query) and [[context relevance]] (evaluating whether the retrieved context is focused and contains minimal irrelevant information). By isolating these three aspects, developers can pinpoint whether a failure in a RAG pipeline stems from the retrieval component or the generation component.

# Dataset / Experimental Setup

The experiments in the paper evaluate the proposed [[Ragas]] framework using the `gpt-3.5-turbo-16k` model via the OpenAI API to execute the evaluation prompts. Since the framework is designed for [[reference-free evaluation]], it does not require standard labeled datasets with ground truth answers. The setup focuses on validating how well the automated LLM-based scores correlate with human judgments across different RAG configurations.

# Evaluation Metrics

- **[[faithfulness]]**: Measures whether the claims made in the generated answer can be directly inferred from the retrieved context, used to detect [[hallucination detection|hallucinations]].
- **Answer Relevance**: Assesses how well the generated answer addresses the user's initial query.
- **[[context relevance]]**: Evaluates whether the retrieved context is focused and contains minimal irrelevant information, which is crucial for managing LLM context window costs and performance.
- **[[perplexity]]**: Mentioned as a traditional baseline metric that [[Ragas]] aims to improve upon or bypass due to its limitations with closed-source models.

# Key Results

- Developed and validated [[Ragas]], a fully automated, [[reference-free evaluation]] framework for RAG pipelines.
- Demonstrated that LLM-assisted prompting can successfully decompose answers into atomic statements and verify them against retrieved contexts to compute a reliable [[faithfulness]] score.
- Established a workflow that integrates seamlessly with popular LLM development tools like [[llama-index]] and [[Langchain]], facilitating rapid prototyping and optimization.

# Limitations

The evaluation framework relies heavily on the prompting of a powerful LLM (like `gpt-3.5-turbo-16k`), making it sensitive to prompt design and the inherent biases or API limitations of the underlying evaluator model. Additionally, because it is reference-free, it may struggle to detect subtle semantic errors if the evaluator LLM itself fails to reason correctly over the context.
