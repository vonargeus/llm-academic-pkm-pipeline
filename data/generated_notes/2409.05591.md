# Summary

MemoRAG is a novel [[Retrieval-Augmented Generation]] (RAG) framework designed to overcome the limitations of traditional RAG when processing long, unstructured contexts. It employs a [[dual-system architecture]] inspired by human cognition: a lightweight [[global memory]] system first skims the entire context to form a compressed representation, while a heavy expressive system generates the final response. The core finding is that by generating "retrieval clues" (draft answers) from this global memory, the system can effectively navigate complex tasks where search intent is implicit, significantly outperforming both standard RAG and long-context LLMs.

# Main Contribution

- Proposed **MemoRAG**, a framework that shifts RAG from a sequential "retrieve-then-generate" pipeline to a memory-informed "recall-retrieve-generate" process.
- Introduced a **dual-system architecture** that balances efficiency and expressiveness by separating long-range memory formation from high-quality answer generation.
- Developed a **KV compression** mechanism for the memory module, allowing for flexible and scalable context handling.
- Implemented **RLGF (Reinforcement Learning from Generation quality’s Feedback)**, a novel optimization algorithm that trains the memory module to generate more effective retrieval clues based on the final output quality.

# Research Problem

Traditional [[Retrieval-Augmented Generation]] (RAG) methods struggle with [[Long Context Processing]] because they rely on two assumptions: that queries are explicitly stated and that the knowledge base is well-structured (e.g., Wikipedia). In many real-world scenarios, such as summarizing a 100-page document or analyzing character relationships in a novel, the search intent is implicit and the context is [[unstructured context]]. This creates a gap where standard retrievers fail to locate relevant information because they lack a "big picture" understanding of the document, while full-context LLMs remain too computationally expensive for widespread use.

# Method

MemoRAG utilizes a [[dual-system architecture]] consisting of a **Memory Module** (System 1) and a **Generation Module** (System 2). The Memory Module is a lightweight LLM optimized for long-range processing using [[KV compression]]. When a document is ingested, this module creates a [[global memory]] by compressing the context into a fixed number of latent tokens. When a task is presented, the Memory Module does not retrieve immediately; instead, it generates **retrieval clues**—essentially "draft answers" or internal hypotheses about where the information might be.

These clues serve as high-quality queries for the retrieval tool, which then extracts specific evidence from the original, uncompressed context. Finally, the heavy **Generation Module** (e.g., a larger LLM like Llama-3) synthesizes the retrieved evidence and the initial clues to produce the final answer. To ensure the memory module provides useful clues, the authors use **RLGF (Reinforcement Learning from Generation quality’s Feedback)**, which rewards the memory module when its clues lead to a high-quality final response from the generator.

# Dataset / Experimental Setup

The system was evaluated using two major long-context benchmarks: [[LongBench]] and [[InfiniteBench]]. These include QA tasks (e.g., HotPotQA) and non-QA tasks (e.g., summarization of government reports). Additionally, the authors curated a **long-document understanding benchmark** covering 20 diverse domains including law, finance, and programming. The experimental setup compared MemoRAG against standard RAG, [[HyDE]], [[GraphRAG]], and native long-context models like Mistral-7B-32K.

# Evaluation Metrics

- **F1-score**: Used for multi-hop QA and short-answer tasks to measure token-level overlap.
- **Accuracy**: Used for classification and specific fact-retrieval tasks.
- **ROUGE-L**: Used for summarization tasks to evaluate the quality of long-form generation.

# Key Results

- **Superiority over Standard RAG**: MemoRAG significantly outperformed traditional RAG on non-QA tasks where search intent is not explicitly stated in the query.
- **Comparison with Advanced Methods**: It surpassed specialized RAG variants like [[HyDE]] and [[GraphRAG]] by providing more context-aware retrieval clues.
- **Efficiency**: MemoRAG demonstrated competitive inference speeds and lower GPU memory costs compared to processing the full context with heavy LLMs, thanks to the [[KV compression]] in the light memory system.
- **Performance vs. Long LLMs**: In several benchmarks, MemoRAG achieved higher accuracy than models that could natively fit the entire context in their window, suggesting that the "recall-then-retrieve" mechanism reduces noise and "lost-in-the-middle" effects.

# Limitations

The paper notes that the performance of MemoRAG is heavily dependent on the quality of the [[KV compression]] and the initial training of the memory module. While [[RLGF]] helps, the memory module may still suffer from "hallucinated clues" if the compression ratio is too high or the context is extremely dense with technical jargon. Additionally, the dual-system approach requires maintaining two models, which might increase deployment complexity.
