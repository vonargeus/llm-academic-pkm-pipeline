# Summary

[[DINGO]] is a novel constrained decoding strategy designed specifically for [[diffusion LLMs]], which generate tokens in parallel rather than sequentially. Unlike traditional autoregressive models, diffusion models lack native mechanisms to enforce formal constraints, making them unreliable for structured tasks like [[JSON generation]]. By employing a [[dynamic programming]]-based approach, [[DINGO]] ensures that generated outputs strictly adhere to user-specified [[regular expressions]] while preserving the model's original probability distribution. The method achieves significant performance gains on symbolic reasoning and structured output benchmarks without the computational overhead of resampling.

# Main Contribution

- Introduces [[DINGO]], the first constrained decoding algorithm for [[diffusion LLMs]] that supports arbitrary [[regular expressions]].
- Utilizes a [[dynamic programming]] approach to ensure the generated output maximizes probability under the model's distribution while maintaining structural validity.
- Provides a provably [[distribution preservation|distribution-preserving]] inference mechanism that avoids computationally expensive resampling techniques.

# Research Problem

Existing [[diffusion LLMs]] struggle to provably enforce formal constraints, such as [[regular expressions]], which are critical for tasks requiring [[structured output generation]]. Because these models generate blocks of tokens in parallel, traditional constrained decoding algorithms—which rely on sequential, left-to-right token prediction—are fundamentally incompatible. This gap leaves diffusion models unreliable for downstream applications that require strict syntactic correctness, such as symbolic solvers or fixed-schema data generation.

# Method

[[DINGO]] addresses the challenge of constrained inference in [[diffusion LLMs]] by treating the output generation as a search for the highest-probability string that satisfies a given [[regular expression]]. Since diffusion models iteratively unmask tokens, [[DINGO]] restricts token choices at each diffusion step to ensure the final output remains a valid prefix of the target language.

The core of the method is a [[dynamic programming]] algorithm that operates over the state space of the [[DFA]] representing the target [[regular expression]]. By calculating the maximum probability path through the DFA, [[DINGO]] ensures that the unmasking process converges to a structurally valid output that is optimal with respect to the model's predicted token distribution. This approach differs from prior work by avoiding greedy, step-wise token selection or costly resampling, ensuring both efficiency and optimality.

# Dataset / Experimental Setup

The authors evaluate [[DINGO]] using two primary benchmarks:

- [[JSON generation benchmark]]: A dataset focused on producing valid, schema-compliant JSON outputs.
Experiments were conducted on open-source [[diffusion LLMs]] to compare the performance of [[DINGO]] against unconstrained inference, measuring the effectiveness of the constraint enforcement.

# Key Results

- [[DINGO]] achieves up to a 68% improvement in performance over unconstrained inference on the [[GSM-symbolic]] and [[JSON generation benchmark]] tasks.
- The method successfully enforces structural constraints without distorting the underlying probability distribution of the [[diffusion LLM]].
- Demonstrates scalability and efficiency compared to resampling-based methods, making it suitable for practical deployment in [[structured output generation]] tasks.

# Limitations

The current formulation of [[DINGO]] focuses on [[regular expressions]], and its extension to more complex grammars (like context-free grammars) is not explicitly detailed. Additionally, the performance may vary depending on the complexity of the target language and the underlying diffusion model's capacity to handle specific structural constraints.
