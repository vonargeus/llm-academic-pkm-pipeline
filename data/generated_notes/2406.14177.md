# Summary

The paper introduces [[SimulSeamless]], a system designed for the [[IWSLT 2024]] [[Simultaneous Speech Translation]] evaluation. The authors solve the problem of adapting large-scale offline [[Speech Foundation Models]] for real-time tasks without the need for expensive retraining or fine-tuning. They achieve this by combining the [[SeamlessM4T]] model with [[AlignAtt]], a [[simultaneous inference]] policy that utilizes [[cross-attention]] scores to manage the [[quality-latency tradeoff]]. The main finding is that this "off-the-shelf" approach yields competitive results across multiple language pairs, supporting over 140 source languages.

# Main Contribution

- Proposed [[SimulSeamless]], a zero-shot framework that enables [[Simultaneous Speech Translation]] using the [[SeamlessM4T]] foundation model.
- Integrated the [[AlignAtt]] policy to handle real-time decision-making based on [[audio-translation alignments]] without model adaptation.
- Demonstrated competitive performance on [[IWSLT 2024]] benchmarks for English to German, Japanese, Chinese, and Czech to English.
- Released an open-source implementation that leverages the massive multilinguality of foundation models for simultaneous tasks.

# Research Problem

The paper addresses the challenge of enabling [[Simultaneous Speech Translation]] (SimulST) using offline-trained [[Speech Foundation Models]]. Traditional SimulST systems often require task-specific retraining or complex architectural modifications to balance translation quality with output latency. This research explores whether high-performance offline models like [[SeamlessM4T]] can be effectively repurposed for simultaneous tasks using non-intrusive policies. This addresses the gap in the literature regarding the efficient deployment of massive [[multimodal models]] in low-latency, real-time scenarios.

# Method

The authors present [[SimulSeamless]], which integrates the [[SeamlessM4T]] (medium configuration) with the [[AlignAtt]] policy. The underlying architecture of [[SeamlessM4T]] consists of a [[W2V-BERT 2.0]] encoder for [[self-supervised speech representations]], a [[Length Adapter]] based on the [[M-adaptor]] for temporal compression via [[attention pooling]], and an [[NLLB]] decoder. 

To enable [[simultaneous inference]], the [[AlignAtt]] policy analyzes [[cross-attention]] scores at each time step to determine if the model has sufficient information to emit a token. It assigns predicted words to specific audio frames; if a word aligns with the most recent (and thus potentially unstable) frames, the system waits for more input. This differs from prior work like [[SeamlessStreaming]], which requires supervised fine-tuning with [[EMMA]]. The authors specifically tuned the decoder layer from which attention scores were extracted to optimize the [[quality-latency tradeoff]].

# Dataset / Experimental Setup

The system was evaluated using the [[IWSLT 2024 Simultaneous Translation Evaluation Campaign data]]. This included the [[MuST-C]] v2.0 tst-COMMON set for [[English-German]], [[English-Japanese]], and [[English-Chinese]], as well as a specific development set for [[Czech-English]]. The experimental setup utilized the [[SeamlessM4T]]-medium checkpoint (1.2B parameters) and 80-dimensional [[Mel filterbanks]] for audio processing. Inference was conducted on a single [[NVIDIA V100]] GPU using the [[SimulEval]] toolkit.

# Evaluation Metrics

- [[BLEU]]: Used to measure translation quality via [[sacreBLEU]] (character-level for Chinese/Japanese, word-level for others).
- [[Average Lagging]] (AL): The primary metric for measuring the time delay in the simultaneous setting.
- [[Length Adaptive Average Lagging]] (LAAL): A latency metric that accounts for differences in target and source lengths.
- [[Average Token Delay]] (ATD): Measures the delay of each generated token.
- Computationally aware metrics: Evaluated to account for the real-world processing time of the 1.2B parameter model.

# Key Results

- [[SimulSeamless]] achieved results comparable to or better than previous years' specialized submissions without any task-specific training.
- For [[English-German]], the system achieved an AL of approximately 2.0s by extracting [[cross-attention]] scores from the 27th or 28th decoder layer.
- The choice of decoder layer significantly impacted performance; for instance, [[English-Japanese]] and [[English-Chinese]] performed best using layers 20-23.
- The system successfully scaled to multiple language pairs (En-De, En-Ja, En-Zh, Cs-En) using the same underlying foundation model.
- Normalizing [[cross-attention]] scores frame-wise was found to be essential to prevent weights from being skewed toward specific frames.

# Limitations

The system's performance is highly dependent on the selection of the specific decoder layer for [[AlignAtt]], which requires manual tuning for different language pairs. Additionally, as an "off-the-shelf" approach, it may not reach the absolute peak performance of models specifically fine-tuned for simultaneous tasks, such as [[SeamlessStreaming]]. The computational cost of running a 1.2B parameter model like [[SeamlessM4T]] also poses challenges for extremely low-latency hardware environments.
