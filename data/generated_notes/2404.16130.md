# Summary

This paper introduces [[GraphRAG]], a novel framework designed to overcome the limitations of conventional [[vector RAG]] when handling global queries that require an understanding of an entire text corpus. The authors solve the problem of "global sensemaking" by using a Large Language Model ([[LLM]]) to construct a hierarchical [[entity knowledge graph]] from raw text, which is then partitioned into modular communities. By generating and indexing [[community summaries]], the system can answer broad, thematic questions through a [[map-reduce]] process. The main finding is that [[GraphRAG]] significantly outperforms traditional RAG in both the [[Comprehensiveness]] and [[Diversity]] of answers for global [[Query-Focused Summarization]] tasks.

# Main Contribution

- Proposed the [[GraphRAG]] framework for global [[sensemaking]] over large-scale private text corpora.
- Developed a hierarchical indexing strategy that uses [[community detection]] (e.g., [[Leiden algorithm]]) to group related entities and generate multi-level [[community summaries]].
- Introduced a [[map-reduce]] query processing architecture that synthesizes global answers from distributed community-level insights.
- Provided an open-source implementation and a novel [[LLM-as-a-judge]] evaluation framework for global summarization without ground-truth answers.

# Research Problem

The paper addresses the failure of conventional [[Retrieval-Augmented Generation]] ([[vector RAG]]) to perform "global sensemaking." While standard RAG excels at retrieving specific, localized facts, it struggles with queries that require synthesizing information across an entire dataset (e.g., "What are the main themes?"). This is essentially a [[Query-Focused Summarization]] (QFS) problem. Existing QFS methods do not scale to the massive token counts typical of modern RAG systems, creating a gap between localized retrieval and scalable global understanding.

# Method

The [[GraphRAG]] approach consists of two primary phases: index construction and query processing. During indexing, the system uses an [[LLM]] to extract an [[entity knowledge graph]] from source documents, identifying nodes (entities) and edges (relationships). This graph is then subjected to [[graph partitioning]] using the [[Leiden algorithm]] to identify nested "communities" of closely related entities. The system then generates [[community summaries]] in a bottom-up, [[hierarchical indexing]] fashion, where higher-level summaries recursively incorporate information from lower-level ones.

For query processing, [[GraphRAG]] employs a [[map-reduce]] strategy. When a global query is received, the "map" step involves using the [[community summaries]] at a specific level of the hierarchy to generate multiple partial responses in parallel. In the "reduce" step, the [[LLM]] aggregates these partial responses into a final, coherent global answer. This differs from [[vector RAG]] by prioritizing the structural and thematic relationships in the data over simple semantic similarity in a vector space.

# Dataset / Experimental Setup

The authors evaluated the system using two representative real-world text datasets in the 1 million token range. The experimental setup compared [[GraphRAG]] against a baseline [[vector RAG]] system. To ensure a fair and robust evaluation of global queries where no ground truth exists, they used an adaptive benchmarking approach. This involved using an [[LLM]] to generate diverse personas and corpus-specific questions, which were then answered by both systems and evaluated by a separate [[LLM-as-a-judge]].

# Evaluation Metrics

- **[[Comprehensiveness]]**: Evaluates how thoroughly the answer covers all aspects of the global query.
- **[[Diversity]]**: Measures the variety of perspectives and themes included in the generated response.
- **[[LLM-as-a-judge]]**: A qualitative comparison where a secondary LLM ranks the quality of answers based on predefined criteria.

# Key Results

- [[GraphRAG]] consistently outperformed [[vector RAG]] on global sensemaking questions.
- In head-to-head comparisons using [[GPT-4]], [[GraphRAG]] showed substantial improvements in [[Comprehensiveness]] and [[Diversity]].
- The hierarchical nature of the [[community summaries]] allowed the system to scale its summaries to different levels of granularity depending on the user's needs.
- The system demonstrated the ability to capture "connections" (people, places, events) that are often missed by localized chunk-based retrieval.

# Limitations

The authors note that the process of building the graph index and generating summaries for every community is computationally expensive and requires a high volume of [[LLM]] tokens. There is also a dependency on the LLM's ability to accurately extract entities and relationships, as errors in the initial graph construction can propagate through the hierarchy.
