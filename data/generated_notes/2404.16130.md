# Summary

This paper introduces [[GraphRAG]], a methodology designed to overcome the limitations of conventional [[vector RAG]] when handling global queries that require an understanding of an entire text corpus. The authors solve this by using a [[Large Language Model]] (LLM) to construct a [[hierarchical indexing|hierarchical graph index]], extracting an [[entity knowledge graph]] and partitioning it into communities. The main finding is that [[GraphRAG]] significantly outperforms baseline RAG systems in [[sensemaking]] tasks, providing more comprehensive and diverse answers for global [[Query-Focused Summarization]] (QFS).

# Main Contribution

- Introduction of [[GraphRAG]] for global [[sensemaking]] over large text corpora.
- A two-stage graph index construction method using LLMs to derive entities and [[community summaries]].
- A [[map-reduce]] approach for generating global answers from hierarchical community summaries.
- An evaluation framework using [[LLM-as-a-judge]] for [[Query-Focused Summarization]] without ground-truth answers.

# Research Problem

Conventional [[Retrieval-Augmented Generation]] (RAG) excels at retrieving specific, localized information but fails on "global" questions that require synthesizing themes across an entire dataset (e.g., "What are the main themes in this corpus?"). This is essentially a [[Query-Focused Summarization]] (QFS) problem. Existing QFS methods do not scale well to the large volumes of text typically indexed by RAG systems, creating a gap between local retrieval and large-scale global [[sensemaking]].

# Method

The [[GraphRAG]] approach consists of two primary phases: indexing and query processing. During indexing, the system uses an LLM to extract an [[entity knowledge graph]] from source documents, identifying entities, relationships, and claims. This graph is then subjected to [[community detection]] algorithms (such as [[Leiden]] or [[Louvain]]) to partition nodes into a hierarchy of nested modular communities. For each community, the LLM generates a [[community summaries|community summary]] in a bottom-up fashion, where higher-level summaries recursively incorporate information from lower-level ones.

At query time, [[GraphRAG]] employs a [[map-reduce]] processing strategy. In the "map" step, the system uses the pre-generated [[community summaries]] from a specific level of the hierarchy to generate partial responses to the user's query in parallel. In the "reduce" step, these partial responses are aggregated and summarized by the LLM to produce a final, comprehensive global answer. This differs from [[vector RAG]] by prioritizing structured thematic summaries over raw text chunk retrieval.

# Dataset / Experimental Setup

The authors evaluated the system using two representative real-world text datasets in the 1 million token range. The experimental setup compared [[GraphRAG]] against a standard [[vector RAG]] baseline. To generate evaluation queries, they used an [[adaptive benchmarking]] approach where an LLM generated diverse personas and associated [[sensemaking]] questions tailored to the specific corpus.

# Evaluation Metrics

- **[[Comprehensiveness]]**: Evaluates how thoroughly the answer covers all aspects of the global query.
- **[[Diversity]]**: Measures the variety of perspectives and themes included in the response.
- **[[LLM-as-a-judge]]**: Uses a high-capacity model (e.g., GPT-4) to compare head-to-head performance between RAG variants based on predefined qualitative criteria.

# Key Results

- [[GraphRAG]] consistently outperformed [[vector RAG]] in both [[comprehensiveness]] and [[diversity]] for global queries.
- The use of [[hierarchical indexing]] allowed the system to provide different levels of detail depending on the community level selected for the [[map-reduce]] step.
- Qualitative results indicated that [[GraphRAG]] provides better "global sensemaking," connecting disparate parts of the corpus that [[vector RAG]] would miss due to its reliance on local semantic similarity.
- The [[LLM-generated knowledge graph]] proved robust enough to support complex reasoning even without manual curation.

# Limitations

The primary limitations include the high computational cost and latency associated with the initial graph construction and [[community summarization]] phases, as they require multiple LLM passes. Additionally, the system's performance is heavily dependent on the quality of the underlying LLM used for entity extraction and summarization. There is also a potential for [[hallucination]] if the LLM incorrectly identifies relationships during the graph construction phase.
