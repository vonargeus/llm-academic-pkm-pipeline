# Summary

[[RAPTOR]] addresses the limitations of traditional [[Retrieval-Augmented Generation]] (RAG) systems that rely on retrieving short, contiguous text chunks, which often lack holistic document context. The authors solve this by recursively clustering and summarizing text segments to build a hierarchical tree structure, representing the document at multiple levels of abstraction. The main finding is that this tree-organized retrieval significantly outperforms flat retrieval methods on complex, thematic question-answering tasks, achieving a 20% improvement in [[absolute accuracy]] on the [[QuALITY]] benchmark when paired with [[GPT-4]].

# Main Contribution

- Introduction of a novel indexing and retrieval algorithm that uses [[recursive summarization]] to build a hierarchical tree from the bottom up.
- A [[clustering]]-based approach to group semantically related text segments regardless of their original position in the document.
- Demonstration of state-of-the-art performance on long-context and complex reasoning datasets including [[NarrativeQA]], [[QASPER]], and [[QuALITY]].

# Research Problem

Existing [[Retrieval-Augmented Generation]] models typically retrieve only a few short, contiguous text chunks from a corpus. This approach fails to capture the [[discourse structure]] and global context of long documents, making it difficult for [[Large Language Models]] to answer thematic questions that require integrating information from disparate parts of a text. This gap in the literature limits the effectiveness of RAG for tasks involving complex, multi-step reasoning over lengthy narratives or technical papers.

# Method

The [[RAPTOR]] method constructs a tree-structured index through a multi-step process. First, the source text is segmented into 100-token leaf nodes and converted into [[vector embedding]] representations using [[SBERT]]. These nodes are then grouped using a soft [[clustering]] algorithm, allowing individual chunks to belong to multiple clusters to preserve context. A [[Large Language Models|Large Language Model]] then generates an [[abstractive processing|abstractive summary]] for each cluster. These summaries are re-embedded and the process is repeated recursively, creating higher-level nodes until a root is reached. 

At inference time, [[RAPTOR]] offers two retrieval strategies: **Tree Traversal**, which prunes the tree level-by-level to find relevant nodes, and **Collapsed Tree**, which treats all nodes across all layers as a flat pool for retrieval. This allows the system to retrieve both granular details from leaf nodes and high-level thematic summaries from parent nodes, providing a richer context for the LLM.

# Dataset / Experimental Setup

The system was evaluated on three major benchmarks:
- [[NarrativeQA]]: Free-text responses based on full-length books and movie scripts.
- [[QASPER]]: Question answering based on full-text scientific papers.
- [[QuALITY]]: Multiple-choice questions requiring reasoning over medium-length passages.

The experimental setup involved testing [[RAPTOR]] with different base models, including [[UnifiedQA]], [[GPT-3]], and [[GPT-4]]. The leaf nodes were embedded using the `multi-qa-mpnet-base-cos-v1` [[SBERT]] model.

# Evaluation Metrics

- [[absolute accuracy]]: Used primarily for the [[QuALITY]] multiple-choice benchmark to measure the percentage of correct answers.
- **F1 Score / Exact Match**: Standard metrics for open-ended question answering in [[NarrativeQA]] and [[QASPER]] to evaluate the overlap between generated and ground-truth answers.

# Key Results

- **State-of-the-Art Performance**: [[RAPTOR]] coupled with [[GPT-4]] achieved a new SOTA on [[QuALITY]] with a 20% absolute improvement over previous methods.
- **Thematic Retrieval**: On [[NarrativeQA]], [[RAPTOR]] significantly outperformed traditional RAG, demonstrating its ability to handle questions about overall plot and character development.
- **Scalability**: The tree construction process scales linearly with the number of tokens, making it viable for large-scale [[Information Retrieval]].
- **Retrieval Strategy**: The "Collapsed Tree" approach generally outperformed "Tree Traversal," suggesting that searching across all levels of abstraction simultaneously is more effective for complex queries.

# Limitations

The authors note that while [[recursive summarization]] captures high-level themes, the process can be computationally more expensive than flat indexing due to the LLM calls required for summarization. Additionally, the quality of the tree is dependent on the summarization capabilities of the underlying [[Large Language Models]], and lossy compression during summarization could potentially omit specific granular details.
