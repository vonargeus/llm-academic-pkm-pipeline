# Summary

[[ALIGNVLM]] introduces a novel architectural component called the [[ALIGN connector]] to improve the integration of visual and textual data in [[Vision-Language Models]]. The authors address the "representation gap" by mapping visual features into a [[weighted average of LLM text embeddings]] rather than using direct linear projections. This approach ensures that visual inputs remain within the [[convex hull]] of the [[LLM]]'s pre-trained [[embedding space]], leveraging existing [[linguistic priors]]. The main finding is that this method significantly improves [[data-efficiency]] and performance on [[Multimodal Document Understanding]] tasks, especially in low-resource scenarios.

# Main Contribution

- Proposed the [[ALIGN connector]], a novel vision-text alignment mechanism that represents visual features as a [[convex combination of text embeddings]].
- Introduced the [[ALIGNVLM]] family of models, which achieves state-of-the-art performance on various document-centric benchmarks.
- Demonstrated that constraining visual features to the LLM's linguistic [[latent space alignment|latent space]] leads to faster convergence and higher robustness to noise.

# Research Problem

Existing [[Vision-Language Models]] typically use [[multilayer perceptrons]] (MLPs) or attention-based connectors to project visual features into an LLM's input space. However, these [[shallow fusion]] methods lack the [[inductive bias]] necessary to ensure that projected visual features align with the linguistic structure the LLM was trained on. This often results in "out-of-distribution" features that require massive amounts of multimodal data to correct. This paper addresses the need for a more data-efficient and semantically grounded way to bridge the gap between [[Computer Vision]] and [[Natural Language Processing]] modalities.

# Method

The core of the [[ALIGNVLM]] approach is the [[ALIGN connector]]. Instead of projecting visual features from a [[vision encoder]] (like [[CLIP]] or [[SigLIP]]) directly into a continuous vector space, the model maps them to a probability distribution over the LLM's existing vocabulary. This is achieved by passing the visual features through a linear layer followed by a [[softmax]] function, where the output dimensions match the size of the LLM's vocabulary.

The final representation for each visual token is computed as a [[weighted average of LLM text embeddings]] based on this distribution. By forcing the visual features to be a [[convex combination of text embeddings]], the method ensures that the visual input resides within the [[convex hull]] of the LLM's pre-trained [[embedding space]]. This design choice acts as a regularizer, preventing the model from generating noisy or uninterpretable representations for the LLM.

This approach differs from prior work like [[Ovis]], which learns a completely new set of visual embeddings, or [[LLaVA]], which uses unconstrained [[multilayer perceptrons]]. By reusing the LLM's own embedding matrix, [[ALIGNVLM]] maintains a lower parameter count while inheriting the [[linguistic priors]] of the language model, facilitating better [[cross-modal interaction]] without modifying the internal weights of the LLM during initial alignment.

# Dataset / Experimental Setup

The authors evaluated [[ALIGNVLM]] across a wide range of [[Multimodal Document Understanding]] benchmarks. These include [[DocVQA]] and [[InfoVQA]] for document and infographic question answering, [[DeepForm]] and [[KLC]] for form and key-value extraction, and [[WTQ]], [[TabFact]], and [[TableVQA]] for table-based reasoning. They also included [[ChartQA]] and [[TextVQA]] to test general visual text recognition. The experiments compared the [[ALIGN connector]] against [[MLP]], [[Perceiver Resampler]], and [[Ovis]] connectors using a unified training setup with the [[Llama-3.2-3B]] backbone to ensure a fair comparison.

# Evaluation Metrics

- [[Accuracy]]: Used for tasks requiring exact matches, such as [[TabFact]] and [[WTQ]].
- [[ANLS]] (Average Normalized Levenshtein Similarity): The primary metric for [[DocVQA]], [[ChartQA]], and [[InfoVQA]], used to evaluate the quality of extracted text while allowing for minor OCR or spelling discrepancies.

# Key Results

- [[ALIGNVLM]] outperformed all other connector types across the nine evaluated benchmarks. For instance, on [[DocVQA]], it achieved an [[ANLS]] of 79.63 compared to 74.68 for the [[MLP]] baseline.
- In [[ChartQA]], the model reached 71.88, significantly higher than the [[Perceiver Resampler]] (65.16) and [[MLP]] (67.92).
- The model demonstrated superior [[data-efficiency]]; when trained on only 10% of the data, [[ALIGNVLM]] maintained a much higher performance lead over [[MLP]]-based models than it did at 100% data.
- Qualitative analysis showed that the [[ALIGN connector]] effectively maps visual regions (like a "Total" field in an invoice) to semantically relevant text tokens in the LLM's vocabulary, confirming the intended [[latent space alignment]].

# Limitations

The paper does not explicitly detail extensive failure modes, but the approach's reliance on the [[convex hull]] of text embeddings suggests that its performance is inherently capped by the richness of the LLM's pre-trained [[embedding space]]. If the LLM's vocabulary or embeddings are poorly trained for specific domains, the [[ALIGN connector]] may struggle to represent those visual concepts accurately. Additionally, the computational overhead of performing a weighted sum over a very large vocabulary (e.g., 128k tokens) could be a factor in extremely latency-sensitive applications.
