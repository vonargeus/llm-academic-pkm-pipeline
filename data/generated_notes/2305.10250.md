# Summary

This paper addresses the limitation of [[Large Language Models]] (LLMs) lacking a [[long-term memory]] mechanism, which restricts their effectiveness in sustained interaction scenarios like personal companionship and psychological counseling. To solve this, the authors introduce [[MemoryBank]], a novel memory mechanism that enables LLMs to store, retrieve, and update memories while dynamically modeling user personalities. Inspired by the [[Ebbinghaus Forgetting Curve]], the system incorporates a human-like forgetting and reinforcement mechanism to manage memory decay over time. The authors demonstrate its effectiveness through [[SiliconFriend]], an AI companion chatbot that successfully recalls past interactions, exhibits empathy, and adapts to user profiles.

# Main Contribution

- Introduction of [[MemoryBank]], a novel human-like [[long-term memory]] mechanism for LLMs based on the [[Ebbinghaus Forgetting Curve]].
- Development of [[SiliconFriend]], an LLM-based AI companion chatbot integrated with [[MemoryBank]] and fine-tuned on [[38k psychological conversations]] to enhance empathy.
- Demonstration of [[MemoryBank]]'s generalizability across both open-source (e.g., [[ChatGLM]], [[BELLE]]) and closed-source (e.g., [[ChatGPT]]) LLMs in bilingual settings.

# Research Problem

Despite the impressive capabilities of [[Large Language Models]], they fundamentally lack a robust [[long-term memory]] mechanism, which limits their utility in scenarios requiring sustained, multi-day human-AI interactions. This deficiency is particularly problematic for applications like personal companionship, [[empathetic dialogue]], and psychological counseling, where building rapport depends on remembering past interactions and understanding user history. Existing LLMs typically rely on static context windows that discard older information once the token limit is exceeded. This paper addresses this gap by designing a dynamic, human-like memory storage and retrieval system that mimics cognitive processes of memory retention and decay.

# Method

The proposed [[MemoryBank]] framework is structured around three central pillars: [[memory storage]], [[memory retrieval]], and [[memory updating]]. The [[memory storage]] acts as a robust data repository that records multi-turn conversations chronologically with timestamps. To prevent context window overflow and mimic human cognition, it distills raw dialogues into a hierarchical structure consisting of daily event summaries, global event summaries, and dynamic [[user profiling]] (personality insights).

The [[memory retrieval]] component extracts context-specific memories relevant to the current user query, which are then appended to the LLM's prompt as augmented context. Crucially, the [[memory updating]] mechanism is inspired by the [[Ebbinghaus Forgetting Curve]] theory. This mechanism dynamically adjusts the "memory strength" of stored events based on the time elapsed and their relative significance, allowing the system to selectively forget trivial details while reinforcing important memories through repeated recall.

To demonstrate this mechanism, the authors developed [[SiliconFriend]], an AI companion chatbot. [[SiliconFriend]] integrates [[MemoryBank]] with LLMs like [[ChatGPT]], [[ChatGLM]], and [[BELLE]]. It is specifically fine-tuned on a dataset of [[38k psychological conversations]] to exhibit heightened empathy, active listening, and guidance capabilities, distinguishing it from standard conversational agents that lack emotional depth and long-term context.

# Dataset / Experimental Setup

The authors utilized a dataset of [[38k psychological conversations]] collected from various online counseling platforms to fine-tune the open-source models ([[ChatGLM]] and [[BELLE]]) for empathetic interaction. For evaluation, the experimental setup combined qualitative real-world user dialogues with a quantitative simulation framework. The quantitative setup involved creating a simulated memory storage containing 10 days of conversations across diverse topics. This simulation featured 15 virtual users with distinct personalities, simulated by [[ChatGPT]], who generated long-term dialogue contexts. From these simulated histories, an evaluation set of 194 probing questions was designed to test memory recall and personality understanding.

# Evaluation Metrics

- **Memory Recall Accuracy**: Evaluates the model's ability to successfully retrieve and reference specific historical facts from past days.
- **Empathy and Companionship Quality**: Assesses the emotional appropriateness and supportiveness of the chatbot's responses in sensitive contexts.
- **User Portrait Accuracy**: Measures how accurately the model summarizes and adapts to the user's simulated personality traits over time.

# Key Results

- [[SiliconFriend]] successfully demonstrated long-term companionship capabilities by recalling relevant past events across a 10-day simulated conversational history.
- The integration of [[MemoryBank]] allowed both open-source ([[ChatGLM]], [[BELLE]]) and closed-source ([[ChatGPT]]) models to maintain consistent [[user profiling]] and adapt their tone to the user's personality.
- Quantitative probing using 194 questions confirmed that the [[Ebbinghaus Forgetting Curve]]-based [[memory updating]] successfully prioritized significant events while naturally decaying less critical conversational details over time.
- Fine-tuning on [[38k psychological conversations]] significantly improved the empathy and discernment of the chatbot compared to base LLM configurations.

# Limitations

The evaluation relies heavily on simulated dialogues generated by [[ChatGPT]] rather than extensive clinical trials with real-world psychological patients. Additionally, the forgetting curve parameters may require manual tuning for different user preferences, and the computational overhead of hierarchical summarization and continuous vector retrieval could limit real-time deployment efficiency.
