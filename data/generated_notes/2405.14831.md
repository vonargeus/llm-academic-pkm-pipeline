# Summary

HippoRAG is a novel retrieval framework designed to provide [[Large Language Models]] (LLMs) with a neurobiologically inspired [[Long-Term Memory]]. It addresses the inability of standard [[Retrieval-Augmented Generation]] (RAG) to integrate knowledge across disparate passages by mimicking the human brain's [[hippocampal indexing theory]]. By combining LLM-based [[Knowledge Graph (KG) construction]] with the [[Personalized PageRank]] (PPR) algorithm, HippoRAG enables efficient [[multi-hop reasoning]] in a single retrieval step. The authors find that HippoRAG significantly outperforms existing RAG methods on complex QA tasks while being substantially faster and cheaper than iterative retrieval techniques.

# Main Contribution

- Introduction of **HippoRAG**, a RAG framework that mimics the interaction between the [[neocortex]] and the hippocampus for human-like [[associative memory]].
- A synergistic orchestration of LLMs for [[LLM-based Information Extraction]], [[schemaless knowledge graph]] structures, and the [[Personalized PageRank]] algorithm for graph-based retrieval.
- Implementation of single-step [[multi-hop reasoning]] that achieves superior performance compared to state-of-the-art iterative methods like [[IRCoT]].
- Significant reduction in computational overhead, demonstrating a 10-20x cost reduction and 6-13x speedup over iterative retrieval baselines.

# Research Problem

Standard [[Retrieval-Augmented Generation]] (RAG) systems typically encode text passages into isolated vector embeddings, which prevents them from effectively integrating knowledge that spans across passage boundaries. This limitation makes it difficult for LLMs to solve [[multi-hop reasoning]] tasks or "path-finding" questions where the connection between entities is not explicitly stated in a single document. While iterative retrieval methods attempt to solve this, they are often computationally expensive, slow, and prone to error propagation, leaving a gap for a more efficient, integrated long-term memory solution.

# Method

HippoRAG operates through two main phases: [[offline indexing]] and [[online retrieval]], mirroring the functions of [[pattern separation]] and [[pattern completion]].

1.  **Offline Indexing (Memory Encoding):** An LLM (acting as the [[neocortex]]) processes the corpus using [[LLM-based Information Extraction]] (OpenIE) to extract triples, which are stored in a [[schemaless knowledge graph]] (the hippocampal index). Dense [[retrieval encoders]] (acting as [[parahippocampal regions]]) are used to link similar noun phrases, ensuring the graph is interconnected even when exact wording differs.
2.  **Online Retrieval (Memory Retrieval):** When a query is received, the LLM extracts named entities. These entities are mapped to "query nodes" in the KG. HippoRAG then applies the [[Personalized PageRank]] (PPR) algorithm, using these nodes as seeds. PPR distributes probability across the graph, identifying the most relevant nodes and, by extension, the most relevant passages.

This approach differs from prior work by replacing flat vector similarity with a graph-based topology that allows for [[associative memory]] and single-step traversal of complex knowledge paths.

# Dataset / Experimental Setup

The authors evaluated HippoRAG using three prominent multi-hop QA datasets: [[MuSiQue]], [[2WikiMultiHopQA]], and [[HotpotQA]]. The experimental setup compared HippoRAG against standard RAG (using [[Contriever]] or [[BGE]] embeddings) and iterative retrieval methods like [[IRCoT]]. The system was tested in both single-step and iterative configurations to measure the performance gains and efficiency improvements.

# Evaluation Metrics

- **[[F1]] and [[Exact Match (EM)]]:** Used to measure the accuracy of the final answers generated by the LLM.
- **[[Retrieval performance]] (Recall@k):** Used to evaluate how effectively the system identifies the necessary passages for multi-hop reasoning.
- **[[Computational cost]] and [[Latency]]:** Used to compare the efficiency of HippoRAG against iterative retrieval methods in terms of API costs and processing time.

# Key Results

- **Superior Accuracy:** HippoRAG outperformed state-of-the-art RAG methods by up to 20% on [[MuSiQue]] and [[2WikiMultiHopQA]].
- **Efficiency Gains:** Single-step retrieval with HippoRAG achieved comparable or better results than [[IRCoT]] while being 10-20x cheaper and 6-13x faster.
- **Synergistic Effects:** When integrated into [[IRCoT]], HippoRAG provided additional gains of up to 20%, showing it complements existing iterative strategies.
- **Path-Finding Capability:** Qualitative analysis showed HippoRAG successfully handles [[path-finding multi-hop questions]] (e.g., linking a professor to a research area through multiple intermediate documents) where standard RAG fails.

# Limitations

The framework's performance is heavily dependent on the quality of the initial [[Knowledge Graph (KG) construction]], which relies on the LLM's ability to perform accurate OpenIE. Additionally, the [[offline indexing]] phase can be computationally intensive during the initial graph build. The authors also note that while PPR is efficient, very large-scale graphs may require further optimization for real-time performance.
