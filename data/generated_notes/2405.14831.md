# Summary

HippoRAG is a novel retrieval framework designed to provide [[Large Language Models]] (LLMs) with a neurobiologically inspired [[Long-Term Memory]]. It addresses the inability of standard [[Retrieval-Augmented Generation]] (RAG) to integrate information across disparate document boundaries by mimicking the human brain's [[hippocampal indexing theory]]. By combining [[LLM-based schemaless KG indexing]] with the [[Personalized PageRank]] algorithm, HippoRAG enables efficient [[multi-hop reasoning]] in a single retrieval step. The main finding is that HippoRAG significantly outperforms existing RAG methods in accuracy while being drastically faster and cheaper than iterative retrieval techniques.

# Main Contribution

- Introduction of HippoRAG, a retrieval framework that mimics the interaction between the [[neocortex]] and [[hippocampus]] for memory integration.
- A novel system design that orchestrates LLMs for [[Knowledge Graph (KG) construction]] and [[Personalized PageRank]] for graph-based retrieval.
- Achieving state-of-the-art performance on [[multi-hop reasoning]] tasks using single-step retrieval.
- Demonstrating a 10-20x reduction in computational cost and 6-13x increase in speed compared to iterative retrieval methods like [[IRCoT]].

# Research Problem

The paper addresses the limitation of current [[Retrieval-Augmented Generation]] (RAG) systems in performing [[knowledge integration]] across multiple passages. While LLMs excel at local reasoning, standard RAG methods encode passages in isolation, failing to capture the relationships between them unless they are explicitly mentioned together. This makes [[multi-hop reasoning]] difficult and expensive, typically requiring multiple iterations of retrieval and generation. The authors aim to bridge this gap by creating a more efficient, [[associative memory]] system that can link information across a corpus without exhaustive iterative processing.

# Method

HippoRAG is structured around three components mirroring human memory:
1.  **The Neocortex (LLM):** During offline indexing, a powerful LLM performs [[LLM-based schemaless KG indexing]] (OpenIE) to extract triples from the corpus. During online retrieval, it extracts named entities from the query.
2.  **The Hippocampal Index ([[schemaless knowledge graph]]):** The extracted triples form a global graph where nodes are noun phrases and edges represent relationships. This graph acts as the associative index.
3.  **The Parahippocampal Regions ([[Retrieval Encoders]]):** Dense encoders link query entities to KG nodes, handling [[pattern separation]] and [[pattern completion]] by resolving synonyms and related concepts.

The retrieval process uses [[Personalized PageRank]] (PPR). The query entities act as "seed" nodes, and PPR distributes probability across the KG. Nodes (and their corresponding original passages) that receive the highest probability are retrieved. This allows the system to find "paths" between distant pieces of information in a single step, effectively performing [[path-finding multi-hop QA]].

# Dataset / Experimental Setup

The framework was evaluated on three complex [[multi-hop reasoning]] datasets: [[MuSiQue]], [[2WikiMultiHopQA]], and [[HotpotQA]]. The experimental setup compared HippoRAG against standard dense retrieval (DR), BM25, and the iterative [[IRCoT]] method. The authors used GPT-3.5 Turbo and Llama-3 for the [[neocortex]] functions and various [[Retrieval Encoders]] (like GTR) for the [[parahippocampal regions]].

# Evaluation Metrics

- [[F1]]: Used to measure the accuracy of both the retrieved passages and the final generated answers.
- [[Retrieval Speed]]: Measured in seconds per query to demonstrate the efficiency of the single-step graph approach.
- [[Retrieval Cost]]: Calculated based on the number of LLM tokens/calls required, highlighting the economic advantage over iterative methods.

# Key Results

- **Performance:** HippoRAG outperformed the best non-iterative RAG methods by up to 20% on [[MuSiQue]] and [[2WikiMultiHopQA]].
- **Efficiency:** Single-step HippoRAG achieved results comparable to or better than [[IRCoT]] while being 10-20 times cheaper and 6-13 times faster.
- **Synergy:** When combined with [[IRCoT]], HippoRAG provided further gains, showing that graph-based retrieval and iterative reasoning are complementary.
- **Robustness:** The method showed superior performance in [[path-finding multi-hop QA]] scenarios where traditional RAG fails to find the initial "bridge" entity.

# Limitations

The performance of HippoRAG is dependent on the quality of the initial [[Knowledge Graph (KG) construction]]; if the LLM fails to extract key triples during indexing, the path-finding capability is diminished. Additionally, the current implementation relies on a [[schemaless knowledge graph]], which, while flexible, may introduce noise if the [[Retrieval Encoders]] incorrectly link unrelated concepts.
