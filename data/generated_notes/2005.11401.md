# Summary

This paper introduces [[Retrieval-Augmented Generation]] (RAG), a general-purpose fine-tuning framework that combines pre-trained [[parametric memory]] (a sequence-to-sequence model) with [[non-parametric memory]] (a dense vector index of Wikipedia accessed via a neural retriever). The authors propose two formulations, [[RAG-Sequence]] and [[RAG-Token]], which treat retrieved documents as latent variables and marginalize over them in different ways. By combining these components, the model can dynamically retrieve relevant documents to generate more factual, specific, and diverse text. RAG achieves state-of-the-art results on several knowledge-intensive tasks, including open-domain [[Question Answering]], while allowing the external knowledge base to be easily swapped or updated without retraining the generator.

# Main Contribution

- Introduced [[Retrieval-Augmented Generation]] (RAG), a novel end-to-end differentiable framework combining pre-trained [[parametric memory]] and [[non-parametric memory]] for [[sequence-to-sequence]] generation.
- Proposed two distinct probabilistic formulations for marginalizing over retrieved documents: [[RAG-Sequence]] (using the same document for the entire sequence) and [[RAG-Token]] (allowing different documents to inform different tokens).
- Demonstrated state-of-the-art performance on multiple open-domain [[Question Answering]] benchmarks, outperforming both purely parametric models and traditional retrieve-and-extract architectures.
- Showed that the non-parametric memory index can be swapped dynamically at test time to update the model's knowledge base without requiring parameter retraining.

# Research Problem

While large pre-trained language models excel at storing factual knowledge implicitly within their parameters, they struggle to access, precisely manipulate, and update this knowledge dynamically. This limitation leads to issues such as factual hallucinations, lack of interpretability or provenance for decisions, and difficulty in updating world knowledge without expensive retraining. Prior hybrid models combining parametric and non-parametric memory, such as [[REALM]] and [[ORQA]], were restricted to extractive downstream tasks. This paper addresses the gap of extending retrieval-augmented architectures to general-purpose [[sequence-to-sequence]] generation for knowledge-intensive NLP tasks.

# Method

The proposed [[Retrieval-Augmented Generation]] (RAG) framework combines a neural retriever with a sequence-to-sequence generator in an end-to-end differentiable model. The retriever component is based on the [[Dense Passage Retriever]] (DPR), which uses a bi-encoder architecture with two [[BERT]]-base models: a query encoder to embed the input query and a document encoder to embed the external document corpus. The document index is stored as a [[dense vector index]] and queried using [[Maximum Inner Product Search]] (MIPS) to retrieve the top-$K$ documents. The generator component is [[BART]]-large, a pre-trained sequence-to-sequence transformer, which takes the concatenation of the input query and the retrieved documents to generate the target sequence.

To train the system end-to-end without direct supervision on which documents to retrieve, the retrieved documents are treated as a [[latent variable]] and marginalized out. The authors propose two formulations for this marginalization:
1. **[[RAG-Sequence]]**: This model assumes the same retrieved document is responsible for generating the entire target sequence. It marginalizes the sequence probability across the top-$K$ documents.
2. **[[RAG-Token]]**: This model allows different retrieved documents to inform different tokens in the generated sequence. It marginalizes the probability of each individual token across the top-$K$ documents, allowing the generator to synthesize information from multiple sources.

During training, the document encoder and its index are kept fixed to avoid the high computational cost of periodic index updates, while the query encoder and the generator are jointly fine-tuned using negative marginal log-likelihood.

# Dataset / Experimental Setup

The non-parametric memory is constructed using a [[Wikipedia]] dump from December 2018, split into 21-million 100-word blocks, which are encoded into a [[dense vector index]] using [[Dense Passage Retriever]] (DPR).
The model is evaluated across several knowledge-intensive datasets:
- **Open-Domain QA**: [[Natural Questions]] (NQ), [[TriviaQA]], [[WebQuestions]] (WQ), and [[CuratedTrec]] (CT).
- **Knowledge-Intensive Generation**: [[MS-MARCO]] (NLG track) and [[Jeopardy]] question generation.
- **Fact Verification**: [[FEVER]].

For open-domain QA, the model is evaluated in an unconstrained generation setup, comparing against retrieve-and-extract baselines and parametric-only models like [[T5]] and [[BART]].

# Evaluation Metrics

- **Exact Match (EM)**: Used for open-domain QA tasks to measure the percentage of generated answers that exactly match the ground truth.
- **BLEU-4**: Used for [[MS-MARCO]] and [[Jeopardy]] generation to evaluate the n-gram overlap and linguistic quality of the generated text.
- **ROUGE-L**: Used for [[MS-MARCO]] to measure the recall-oriented overlap of longest common subsequences.
- **Accuracy**: Used for [[FEVER]] fact verification to measure the correctness of the classification label (Supported/Refuted/NotEnoughInfo).

# Key Results

- **Open-Domain QA**: RAG sets a new state-of-the-art on [[Natural Questions]] (44.5% EM), [[WebQuestions]] (45.5% EM), and [[CuratedTrec]] (60.1% EM), outperforming both parametric-only models (e.g., [[T5]] at 32.6% on NQ) and retrieve-and-extract models. On [[TriviaQA]], RAG achieves 56.8% EM, outperforming contemporary models.
- **Generation Quality**: On [[MS-MARCO]] and [[Jeopardy]] generation, RAG models generate more specific, diverse, and factual language than a parametric-only [[BART]] baseline, as measured by human evaluation and overlap metrics.
- **Fact Verification**: On [[FEVER]], RAG achieves 76.8% label accuracy, which is within 4.3% of state-of-the-art pipeline models that use strong, explicit retrieval supervision.
- **Knowledge Hot-Swapping**: The authors demonstrate that by replacing the December 2018 Wikipedia index with a newer index, RAG can instantly update its world knowledge (e.g., correctly identifying a new world leader) without retraining any parameters.

# Limitations

The model relies on a top-$K$ approximation to marginalize over the latent documents, which may miss relevant documents outside the top-$K$ retrieval pool. Additionally, keeping the document encoder fixed during training prevents the representation of documents from adapting to the generation task, which is a compromise made to avoid the high computational cost of rebuilding the [[dense vector index]] during training.
